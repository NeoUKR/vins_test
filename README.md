# VINS Test

**Українська** | [English](README.en.md)

`VINS Test` — консольна утиліта для перевірки роботи VINS та автоматичного підбору числових параметрів у YAML-конфігурації. Програма підтримує ROS2 (типово) і ROS1, запускає VINS та rosbag, збирає одометрію, оцінює похибку повернення й зберігає результати випробувань у CSV.

Поточну версію та вибраний варіант ROS можна перевірити командою:

```bash
python3 vins_test.py --version
```

## Можливості

- окремий режим спостереження за вже запущеним VINS;
- автоматичний локальний пошук найкращого значення параметра YAML;
- робота з ROS2 і ROS1;
- контроль надходження повідомлень камери, IMU та одометрії;
- контроль загального навантаження CPU та NVIDIA GPU під час випробувань;
- фіксація стабільної початкової позиції (`FIXPOS`);
- повторення кожного випробування й агрегація за медіаною або середнім;
- оцінювання за `Delta XY`, `Delta XYZ` або складеною цільовою функцією;
- резервне копіювання й автоматичне відновлення початкової конфігурації;
- запис усіх результатів у CSV.

Для контролю GPU утиліта використовує `nvidia-smi`. Якщо NVIDIA GPU або ця команда недоступні, випробування продовжується, а показники GPU позначаються як `unavailable` у звіті та залишаються порожніми в CSV.

## Вимоги

- Linux і Python 3;
- встановлений ROS2 або ROS1 із доступними Python-модулями;
- встановлений і зібраний VINS;
- rosbag із топіками камери та IMU;
- YAML-конфігурація VINS із числовим параметром, який потрібно підібрати.

Типові шляхи розраховані на поточну структуру робочих просторів проєкту. Якщо вона відрізняється, передайте власні шляхи через `--ros-setup`, `--vins-setup`, `--config` та інші параметри.

Повний список аргументів:

```bash
python3 vins_test.py --help
```

## Швидкий старт: перевірка VINS

Спочатку запустіть VINS та джерела даних камери й IMU. Потім відкрийте інший термінал і виконайте:

```bash
python3 vins_test.py --test
```

ROS2 використовується типово. Для ROS1 додайте `--ros1`:

```bash
python3 vins_test.py --ros1 --test
```

Типово програма слухає:

- одометрію — `/vins_estimator/odometry`;
- камеру — `/cam0/image_raw`;
- IMU — `/imu0`.

Інші назви можна вказати явно:

```bash
python3 vins_test.py --test \
  --odom-topic /vins_estimator/odometry \
  --cam-topic /camera/image_raw \
  --imu-topic /imu/data
```

Під час сесії відображаються поточна позиція, кількість повідомлень і пройдена відстань. Зупинення VINS, припинення даних IMU або `Ctrl+C` завершує поточну сесію та виводить підсумкові `Delta XY`, `Delta XYZ` і довжину шляху.

Для вимірювання лише після стабілізації позиції використовуйте:

```bash
python3 vins_test.py --test --fixpos \
  --fixpos-time 5 \
  --fixpos-eps 0.02 \
  --fixpos-max-wait 60
```

## Автоматичний підбір параметра

У цьому режимі програма для кожного пробного значення:

1. змінює вибраний параметр у YAML;
2. запускає VINS;
3. відтворює rosbag;
4. збирає одометрію до завершення запису;
5. обчислює цільову оцінку й записує результат у CSV;
6. перевіряє значення з обох боків від поточного найкращого та поступово зменшує крок.

Приклад для ROS2:

```bash
python3 vins_test.py -param td \
  --test-json test.json \
  --config ~/vins_ws/src/VINS-Mono/config/euroc/euroc_config.yaml \
  --bag ~/bags/test/data \
  --ros-setup ~/ros2_jazzy/install/setup.bash \
  --vins-setup ~/vins_ws/install/setup.bash
```

Приклад для ROS1:

```bash
python3 vins_test.py --ros1 -param td \
  --test-json test.json \
  --config ~/vinsmono_ws/src/VINS-Mono/config/euroc/euroc_config.yaml \
  --bag ~/bags/test/data.bag \
  --ros-setup ~/ros1_noetic_ws/install_isolated/setup.bash \
  --vins-setup ~/vinsmono_ws/devel/setup.bash
```

Типові команди запуску:

- ROS2: `ros2 launch vins_estimator euroc.launch.py`;
- ROS1: `roslaunch vins_estimator euroc.launch`.

За потреби команду можна повністю замінити параметром `--launch-cmd`.

## Налаштування `test.json`

Активний параметр завжди вибирається аргументом `-param`. Один JSON-файл може містити налаштування кількох параметрів:

```json
{
  "objective": "composite",
  "delta_xy_weight": 1.0,
  "delta_xyz_weight": 1.0,
  "distance_xy_weight": 0.0,
  "expected_distance_xy": null,
  "defaults": {
    "max_fails": 2,
    "max_trials": 25,
    "max_step_reductions": 4,
    "step_reduce_factor": 2.0
  },
  "parameters": {
    "td": {
      "step": 0.003,
      "min_step": 0.0001
    },
    "acc_n": {
      "step": 0.02,
      "min_step": 0.001,
      "max_trials": 20
    }
  },
  "fixpos": {
    "enabled": true,
    "time": 5.0,
    "eps": 0.02,
    "max_wait": 30.0
  },
  "test": {
    "repeats": 3,
    "aggregation": "median"
  }
}
```

Обов’язкові налаштування кожного параметра після об’єднання з `defaults`:

- `step` — початковий крок пошуку;
- `min_step` — мінімальний крок;
- `max_fails` — допустима кількість невдалих спроб на поточному кроці;
- `max_trials` — максимальна кількість випробувань;
- `max_step_reductions` — максимальна кількість зменшень кроку.

Поле `objective` приймає значення:

- `delta_xy` — мінімізація горизонтальної похибки між початком і кінцем;
- `delta_xyz` — мінімізація просторової похибки;
- `composite` — зважена сума `Delta XY`, `Delta XYZ` і, якщо налаштовано очікувану довжину шляху, похибки `Distance XY`.

Для `test.aggregation` доступні `median` і `mean`.

## Результати та збереження конфігурації

Перед початком підбору поруч із YAML створюється резервна копія такого вигляду:

```text
euroc_config.yaml.backup_vins_tune_YYYYMMDD_HHMMSS
```

Типово початкова конфігурація відновлюється після завершення або зупинення програми. Щоб залишити знайдене найкраще значення в YAML, додайте `--keep-best`.

Результати зберігаються у `vins_param_tune_results.csv`. Інший шлях задається через `--csv`. Для кожного результату CSV та консольний звіт містять мінімальне, максимальне й середнє навантаження CPU/GPU у відсотках.

## Керування виконанням

- `Ctrl+C` — зупинити програму й активні процеси VINS/rosbag;
- `--stop-file PATH` — задати файл безпечного зупинення;
- `--dry-run` — вивести кандидатів без запуску випробувань;
- `--keep-best` — зберегти найкраще знайдене значення;
- `--show-rosbag-output` — показати вивід rosbag;
- `--roslaunch-log-mode quiet|warn|all` — вибрати деталізацію виводу запуску VINS;
- `--no-clock` — не передавати rosbag параметр симуляційного часу.

Типовий файл зупинення — `~/scripts/STOP`. Створити його з іншого термінала можна командою:

```bash
touch ~/scripts/STOP
```

Після виявлення файла програма видалить його, завершить поточну роботу, зупинить дочірні процеси й відновить початкову конфігурацію. Якщо автоматичне видалення не вдалося через права доступу, видаліть файл `STOP` перед наступним запуском.
